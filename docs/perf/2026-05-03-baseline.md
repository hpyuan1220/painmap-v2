# Performance Baseline · 2026-05-03

實作於 main：bundle visualizer 工具 → 字型 self-host → vendor manualChunks。

## Landing first-load (gzipped)

| Asset | Before | After | Δ |
| --- | --- | --- | --- |
| `index-*.js` (entry chunk) | 129,856 B | **70,404 B** | **−45.8 %** |
| `react-vendor-*.js` (new) | — | 60,437 B | +60,437 B (cacheable) |
| `styles-*.css` | 19,335 B | 19,426 B | +91 B (font-face decls) |
| **First-paint critical (entry+vendor+css)** | **149,191 B** | **150,267 B** | +1,076 B (≈0.7 %) |
| **Repeat-visit critical (vendor cache hit)** | **149,191 B** | **89,830 B** | **−39.8 %** |

**取捨**：第一次造訪幾乎打平（多 1 KB gz）；回訪 / 後續部署只動業務 chunk → vendor 走 304，淨省約 60 KB gz。對使用者旅程「來一次寫完 9 卡」是 net-positive。

## Font payload (same-origin self-host)

| File | Size | Source |
| --- | --- | --- |
| `geist-sans-variable.woff2` | 68 KB | npm `geist@1.5.1` → `public/fonts/` |
| `geist-mono-variable.woff2` | 70 KB | npm `geist@1.5.1` → `public/fonts/` |

Variable woff2 + `font-display: swap` + `<link rel="preload">` → no perceptible FOUT on a warm cache; cold cache shows fallback fonts for ≤100 ms then swaps. Previously: 2 jsdelivr stylesheet round-trips before any font byte arrived.

## Vendor chunk policy (`vite.config.ts`)

```
react-vendor    → react, react-dom, scheduler
tanstack-vendor → @tanstack/*  (lazy — only worksheet routes need it)
radix-vendor    → @radix-ui/*  (lazy — only worksheet card forms need it)
icons-vendor    → lucide-react (lazy — only routes that import icons)
```

Landing entry now `import`s only `react-vendor` (verified: 1 sibling chunk reference). Other vendor chunks are loaded lazily by route-level dynamic imports.

## Bundle visualizer

`dist/stats.html` (~1 MB) emitted by `rollup-plugin-visualizer` on every `npm run build`. Reflects the **server (Cloudflare Worker) bundle** — important for staying under Cloudflare Worker size limits. For client-side analysis, use `source-map-explorer dist/client/assets/*.js` ad-hoc.

## What we did NOT do (and why)

| Skipped | Reason |
| --- | --- |
| SSR streaming refactor | TanStack Start defaults already stream |
| Markdown lib dedup (`react-markdown` ↔ `marked`) | `react-markdown` is in `learn.worksheet.07` chunk only — no impact on landing |
| Image optimization | Landing has no large images (Hero is pure CSS) |
| Critical-CSS extraction | 19 KB gz CSS already small enough; tooling cost > benefit |
| Tailwind class purge audit | Tailwind v4 source-aware mode already minimises emit |

## Future bets (not in this round)

1. Switch `react-markdown` → `marked` + `DOMPurify` in `MarkdownView.tsx` (saves ~30 KB gz on Card 7)
2. Pre-render landing as static HTML at build time (requires TanStack Start SSG plugin — currently dev-only experimental)
3. Subset Noto Sans TC to glyphs actually present in landing copy (~80 % size reduction possible, but tooling burden)
